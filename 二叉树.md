# 前、中、后序的递归写法
```
class Solution:
    def preorderTraversal(self, root): # 前序：根 -> 左 -> 右
        if not root: return []
        return [root.val] + self.preorderTraversal(root.left) + self.preorderTraversal(root.right)

    def inorderTraversal(self, root):  # 中序：左 -> 根 -> 右
        if not root: return []
        return self.inorderTraversal(root.left) + [root.val] + self.inorderTraversal(root.right)

    def postorderTraversal(self, root): # 后序：左 -> 右 -> 根
        if not root: return []
        return self.postorderTraversal(root.left) + self.postorderTraversal(root.right) + [root.val]
```
递归写法完全对应了遍历的定义。三种遍历的代码结构几乎一模一样，只是记录值的顺序不同。

# 前、中、后序的递归写法（使用栈）
在面试中，面试官经常要求不使用递归，因为递归在树极深时会导致栈溢出（Stack Overflow）。
## 前序遍历（最简单）
利用栈“先进后出”的特性。为了让左孩子先出栈，我们要先压入右孩子。
```
def preorder(root):
    if not root: return []
    stack, res = [root], []
    while stack:
        node = stack.pop()
        res.append(node.val)
        if node.right: stack.append(node.right) # 右孩子先入
        if node.left: stack.append(node.left)   # 左孩子后入，下次先弹
    return res
```
## 中序遍历（最常用）
模拟“一路向左”直到尽头的过程
```
def inorder(root):
    res, stack, curr = [], [], root
    while curr or stack:
        while curr:          # 一直往左扎到底
            stack.append(curr)
            curr = curr.left
        curr = stack.pop()   # 弹出的就是当前最左的“根”
        res.append(curr.val)
        curr = curr.right    # 转向右子树
    return res
```
## 后序遍历（反转小技巧）
直接写后序迭代较难，但我们可以观察：

前序：根 $\rightarrow$ 左 $\rightarrow$ 右

变形：根 $\rightarrow$ 右 $\rightarrow$ 左

反转变形结果：左 $\rightarrow$ 右 $\rightarrow$ 根（即为后序！）

```
def postorder(root):
    if not root: return []
    stack, res = [root], []
    while stack:
        node = stack.pop()
        res.append(node.val) # 顺序：根 -> 右 -> 左
        if node.left: stack.append(node.left)
        if node.right: stack.append(node.right)
    return res[::-1] # 反转，得到 左 -> 右 -> 根
```

# 二叉树递归
## 第一步：确定递归函数的参数和返回值

不要急着写代码，先问自己两个问题：

参数：我处理当前这个节点，需要外界给我什么信息？（通常就是当前节点 node）。

返回值：我处理完这个节点后，需要给我的“上司”（父节点）汇报什么？

是汇报我的高度？（如：求最大深度）

是汇报我是不是平衡的？（如：平衡二叉树）

还是汇报我已经翻转好的自己？（如：翻转二叉树）

## 第二步：确定终止条件（Base Case）

递归必须有出口，否则会死循环。

对于二叉树，90% 的终止条件都是判断节点是否为空。

```
if not node:
    return 0 # 或者 return None, return True，取决于你的返回值类型
```
## 第三步：确定单层递归的逻辑（最核心）

这是最关键的一步： 想象你现在就站在树的某一个节点上，不要去管你的子节点是怎么算的，默认你的子节点已经帮你把结果算好了。

你只需要思考：利用子节点给你的结果，加上你当前节点的值，怎么拼凑出给父节点的汇报？

### 案例演示 1：求二叉树的最大深度

参数与返回：传入 node，返回以它为根的深度（整数）。

终止条件：if not node: return 0。

单层逻辑：

默认左子树深度 left_h 已算好。

默认右子树深度 right_h 已算好。

我的任务：我是根，我的深度就是 max(left_h, right_h) + 1。

### 案例演示 2：求二叉树的所有节点之和

参数与返回：传入 node，返回该树的总和。

终止条件：if not node: return 0。

单层逻辑：

左边帮我算好了总和 left_sum。

右边帮我算好了总和 right_sum。

我的任务：返回 node.val + left_sum + right_sum。
